6.  Urutan Peserta yang dinyatakan terdaftar mengikuti waktu submisi kegiatan Seminar /

Sidang ( First Come First Serve )

7.  Jika kuota pendaftaran belum terpenuhi, maka jadwal tidak akan dibuat.
8.  Jika Berkas skripsi yang diupload tidak valid, maka pendaftaran akan dibatalkan.

Seminar Proposal

Untuk  maju  seminar  proposal  mahasiswa  perlu  menyelesaikan  dua  submission  di  bawah  ini
secara berurutan.

1. Persetujuan Seminar Proposal

Mahasiswa  diharuskan  mendapatkan  persetujuan  untuk  mengikuti  seminar  Proposal  oleh
dosen pembimbing melalui modul ini.

Mahasiswa wajib mengirimkan file submission berupa :

1.  File Proposal dengan format :  PROPOSAL_NIM.pdf
2.  File  Form  Persetujuan  Seminar  Proposal  yang  sudah  diisi  dan  ditandatangani  oleh
Dosen  Pembimbing  dengan  format  :  DOPING_NIM.pdf beserta dengan lembar kendali
bimbingan pra proposal yang telah ditandatangani oleh dosen pembimbing

Kedua  file  tersebut  diupload  pada  2  file  yang  berbeda  agar  dapat  dilakukan  review  oleh
pembimbing.

2. Workshop Seminar Proposal

Modul  ini  digunakan  untuk  melakukan  penilaian  pada  saat  seminar  proposal.  Mahasiswa
diwajibkan melakukan submission kembali dengan proposal yang sudah siap dengan format
:  Proposal_NIM.pdf

Phase  submission  ditutup  selama  seminar proposal berlangsung dan akan dibuka kembali
setelah seminar proposal selesai.

Seminar Hasil

Untuk  maju  seminar  hasil  mahasiswa  perlu  menyelesaikan  enam  submission  di  bawah  ini
secara berurutan.

1. Uji Program

Uji Program dapat dilakukan secara Offline maupun Online.

Untuk  mahasiswa  yang  melakukan  uji  program  secara  offline,  diharapkan  untuk
mengupload  berkas  penilaian  uji  program  yang  sudah  berisi  nilai dari dosen pembimbing
pada modul submission ini.

Untuk  mahasiswa  yang  melakukan  uji  program  secara  online  dapat  mengikuti  tata  cara
dibawah ini. Tata cara melakukan uji program secara online, yaitu:

1.  Mahasiswa  menghubungi  pembimbing  1  atau  2 untuk menentukan jadwal uji program

dan platform yang akan dipakai untuk presentasi.

2.  Mahasiswa melakukan uji program dan Dosen pembimbing melakukan penilaian sesuai

kriteria pada form penilaian Uji Program.

3.  Setelah selesai melakukan uji program, mahasiswa diwajibkan memberikan submission
pada  modul  ini  dengan  format  :  NIM_Judul  Tugas akhir dan sertakan juga Link Video
Demo Program.

4.  Dosen akan menginput nilai yang telah ia miliki pada submission milik mahasiswa yang

bersangkutan sebagai syarat untuk dapat melanjutkan Seminar Hasil.

2. Plagiarism Check

Seluruh mahasiswa yang akan melakukan seminar hasil, wajib mengumpulkan skripsi yang
sudah  final  agar  dapat  dilakukan  plagiarism check. File yang harus diupload : File Skripsi
Final (Full) dengan nama file NIM_SKRIPSI_CHECK.docx (File word)

Jika  tidak  memenuhi  kriteria  (plagiarisme  melebihi  25%),  maka  dapat  memperbaiki
skripsinya dengan mengacu pada hasil yang dapat dicek di halaman file berikut ini

Untuk  melanjutkan ke tahapan seminar hasil, file akan dicek dan akan dilakukan approval
oleh Tim Check Plagiarism

Pemeriksaan  dilakukan  setiap  Hari  Senin-jumat  di  jam  kerja.  Bagi  yang  telah  memenuhi
kriteria  akan  langsung  di-grade, sementara yang belum memenuhi kriteria silahkan cek di
cek plagiarism ilkom

Seluruh  proses  dilakukan  melalui  elearning  dan  tidak  dibenarkan  menghubungi langsung
melalui Whatsapp atau sejenisnya kepada tim pemeriksa.

3. Form Persetujuan Seminar Hasil

Mahasiswa  wajib  mengisi  form  persetujuan  seminar  hasil  dan  form  tersebut  wajib
ditandatangani oleh dosen pembimbing 1 and 2.

Mahasiswa wajib mengirimkan file submission berupa :

1.  File  Persetujuan  Seminar  Hasil  yang  telah diisi dan ditandatangani oleh Doping 1 & 2

dengan format : PERSETUJUAN_NIM.pdf

2.  Form  lembar  kendali  bimbingan  pra  seminar  hasil  yang  telah  ditandatangani  oleh

Doping 1 & 2.

Form tersedia pada modul form TA

4. Persetujuan Seminar Hasil

Mahasiswa  wajib  mengirimkan  file  submission  berupa  File  Proposal  dengan  format  :
SKRIPSI_NIM.pdf

Dosen  Pembimbing  1  melakukan  approval  pada  submission  mahasiswa  untuk  dapat
memberikan izin melanjutkan ke Seminar Hasil

Approval  submission  ini  ditandai  dengan  kriteria  Nilai  >=50  (Disetujui)  dan  <50  (Tidak
Disetujui)

5. Berkas Berita Acara Seminar Hasil

Mahasiswa  wajib  mengirimkan  file  submission  berupa:  File  Berita  Acara  Seminar  Hasil
dengan format : BASemHas_NIM.doc/docx yang telah terisi kecuali pada bagian nilai. Form
tersedia pada modul form TA

6. Workshop Seminar Hasil

Modul  ini  digunakan  untuk  melakukan  penilaian  pada  saat  Seminar  Hasil.  Mahasiswa
diwajibkan melakukan submission kembali dengan skripsi yang sudah siap dengan format :
SKRIPSI_NIM.pdf.

Phase  submission  ditutup  selama  seminar  hasil  berlangsung  dan  akan  dibuka  kembali
setelah seminar hasil selesai.

Sidang Meja Hijau

Untuk  maju  seminar proposal mahasiswa perlu menyelesaikan empat submission di bawah ini
secara berurutan.

1. Persetujuan Sidang Meja Hijau

Mahasiswa wajib mengirimkan file:

1.  File Proposal dengan format :  SKRIPSI_NIM.pdf
2.  File Persetujuan Sidang Meja Hijau yang telah diisi Nama Doping 1 & 2, Nama Dosen
Pembanding 1 & 2 dengan format : PERSETUJUAN_NIM.pdf dan wajib ditandatangani
oleh dosen pembimbing dan dosen Pembanding.

3.  Form  lembar  kendali  bimbingan  pra  sidang  meja hijau yang telah ditandatangani oleh

Doping 1 & 2.

Dosen  Pembimbing  1  melakukan  approval  pada  submission  ini  untuk  dapat  memberikan
izin  melanjutkan  ke  Sidang  Meja  Hijau  Approval  ditandai  dengan  kriteria  Nilai  >=50
(Disetujui) dan <50 (Tidak Disetujui)

Hijau.

3. Berkas Berita Acara Sidang Meja Hijau

Mahasiswa wajib mengirimkan file submission berupa:

File  Berita  Acara  Sidang  Meja  Hijau  dengan  format  :  BASMH_NIM.doc/docx  yang  telah
terisi kecuali pada bagian nilai (Form tersedia pada modul Form TA).

Bagian yang wajib diisi oleh mahasiswa:

Pada bagian rekapitulasi :

●  Nama Dosen
●  Nama dan Nip Ketua Penguji (Dosen Penguji 1)

pada bagian berita acara sidang meja hijau:

●  nim, nama, judul skripi
●  nama pembimbing 1 dan 2
●  panitia pelaksana ujian sarjana komputer:

○  Ketua (Dosen Penguji 1),
○  Sekertaris (Dosen Penguji 2),
○  Anggota 1 (Dosen Pembimbing 1),
○  Anggota 2 (Dosen Pembimbing 2),
○  Nama Dan Nip Ketua Penguji (Dosen Penguji 1)
○  Nama Dan Nip Ketua Prodi

Jika mahasiswa tidak mengisi bagian seperti yang tersebut diatas maka pengumuman hasil
sidang meja hijau akan ditunda sampai berkas terisi dengan baik seperti disebut diatas.

Setelah  mendapatkan  persetujuan  and  mengupload berkas berita acara mahasiswa wajib
mensubmit  file  final  skripsi  di  modul  Sidang  Meja  Hijau  dibawah  ini  untuk  mendapatkan
jadwal Sidang Meja Hijau

4. Workshop Sidang Meja Hijau

diwajibkan melakukan submission kembali dengan skripsi yang sudah siap dengan format :
SKRIPSI_NIM.pdf.

Phase submission ditutup selama sidang meja hijau berlangsung dan akan dibuka kembali
setelah sidang meja hijau selesai.
